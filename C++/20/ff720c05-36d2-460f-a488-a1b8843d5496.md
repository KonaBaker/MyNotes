# std::format

## usage

格式化字符串

```c++
template< class... Args >
std::string format( std::format_string<Args...> fmt, Args&&... args );
```

基本用法：

```c++
#include <format>
#include <iostream>

int main() {
    std::string name = "Alice";
    int age = 30;
    
    std::string formatted_string = std::format("My name is {} and I am {} years old.", name, age);
    
    std::cout << formatted_string << std::endl;
    return 0;
}
```

关于占位符：{arg-id(optional)} {arg-id(optional) : format-spec}

- {} 默认按顺序索引
- {0} {1} 指定索引

**要么全部指定，要么全部顺序**

- {:.2f}格式规范。



## runtime & compile-time

`std::format`要求格式化字符串需要是编译期常量。

`std::string vformat(std::string_view fmt, std::format_args args);`

- 如果格式串是编译期已知

```c++
auto s = std::format("id={}, name={}", id, name);
```

- 运行时

```c++
// C++20/23
std::string fmt = translate("The answer is {}.");
auto s = std::vformat(fmt, std::make_format_args(42));

// c++26
std::string fmt = translate("The answer is {}.");
auto s = std::format(std::runtime_format(fmt), 42);
```

### c++26

`std::vformat`更像是一种work around **`std::runtime_format`**的出现就是为了解决以下问题：

1) 引用ub

```c++
std::string str = "{}";
std::filesystem::path path = "path/etic/experience";
auto args = std::make_format_args(path.string());
std::string msg = std::vformat(str, args);
```

> 上述问题在P2902R2 C++20中已经修正，make_format_args的参数变成了左值引用

path.string是一个临时变量，make_format_args保存的是引用，会出现ub。

2) 可用/可读

`std::vformat` 是类型擦除的函数，通常用于底层封装，应该not by user。不暴露make_format_args，并且明确表达**运行时字符串**

> 类型擦数：把原本不同类型的信息，统一装进一个“通用表示”里，后续代码不再依赖具体模板实参类型。

**template bloat** 

模板膨胀：因为模板会按不同实参组合不断实例化，导致生成很多重复但相似的代码，增加编译时间、目标文件体积、调试负担。

为了减少模板的实例化数量，就需要使用template bloat

这种类型擦数的函数是底层接口，一般适合封装层的作者。



## chain

`std::format_string`第一个参数接收的是`std::format_string<Args...> fmt`

-> 先调用`basic_format_string<char, type_identity_t<Args>...>` 做格式串合法性检查

-> `fmt.get()` 取出底层的string_view

-> `std::make_format_args` 打包参数

-> `std::vformat(fmt.get(), format_args)`

第一步**basic_format_string**有一个consteval的构造函数

_args args);`

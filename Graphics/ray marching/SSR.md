# SSR

## overview

1.从gbuffer中某个像素点的深度、法线，重建view space坐标。

2.计算view space的反射方向

3.沿着反射方向作ray-march,每一步将view space投影回屏幕，得到uv，对depth buffer进行查询。

- 如果光线深度刚好略大于场景深度，就代表命中，采样颜色返回

- 否则就继续步进，直到超过阈值。

### 加了HIZ

原本ssr在步进的时候每次前进一个像素（mip0）,遇到空旷区域仍然要一步步检测。

现在对depth buffer生成一个"mipmap"，取最小值，即深度小。

先从最高层出发，步进一格，比较射线z和当前格子z。

- 如果射线z小，则没有穿越，整格跳过，步进到下一大格。（未命中）
- 如果射线z大，则表明穿越了这个格子中的某个物体，那么level--。（命中）

这个步进是从格子的边界开始的（防止越过）

### DDA

视空间固定步长有个问题：远处一步可能跨好几个像素，近处一步又不到一个像素。更高效的做法是**在屏幕空间走**，每步推进恰好一个像素（或几个像素）。McGuire 在 2014 年的论文 "Efficient GPU Screen-Space Ray Tracing" 给出了经典做法，核心思路：

1. 把视空间起点 P0 和终点 P1（P0 + reflectDir * maxDistance）都投影到屏幕
2. 在屏幕空间做 DDA，沿着屏幕直线一像素一像素走
3. 同时用透视正确插值出当前点的视空间深度（用 1/z 做线性插值）
4. 拿插值出的深度跟深度图比

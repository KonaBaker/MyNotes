#### deferred decals

使用延迟渲染管线，gbuffer和dbuffer

dbuffer专门存储贴花的信息，比如贴花纹理的漫反射、材质、法线贴图等。

通过两个pass渲染得到gbuffer和dbuffer,最后到lighting pass,传入两个buffer的信息进行颜色混合。

在dbuffer进行渲染的时候，需要禁用深度写入，只用gbuffer中=存储的深度进行比较，确保贴花应用与正确的表面

**注意**：

- **贴花体积**定义了贴花能够影响的3D空间范围.

以子弹来进行举例：

**创建贴花体积**在子弹击中点周围创建一个立方体，其决定贴花影响范围和贴花的朝向。

**渲染体积到屏幕空间** 将立方体作为普通几何体渲染，不是渲染立方体本身，而是渲染它覆盖的屏幕区域。

**像素判断** 检查像素对应点是否在立方体内部，如果在应用dbuffer进行渲染。

每个贴花都使用相同的简单的立方体几何体，只是变换矩阵不同。



- 所有贴花是共享一个dbuffer的

```
第1步：场景渲染
所有几何体 → GBuffer
（位置、法线、基础材质等）

第2步：DBuffer初始化
清空DBuffer为透明
（Alpha = 0，表示无贴花影响）

第3步：贴花累积渲染
for (每个贴花) {
    渲染贴花体积 → 累积到DBuffer
    使用加法混合或Alpha混合
}

第4步：合并阶段
DBuffer + GBuffer → 修改后的GBuffer

第5步：光照计算
使用修改后的GBuffer进行光照
```

dbuffer是像素级的，判断也是，对于多个贴花，同一个像素（可能被多个贴花影响），那么在渲染dbuffer的时候会判断多次，是否每贴花n所影响。

如果影响才会累计到该像素的dbuffer中。

> 多个dbuffer会占用很多内存



- 对于贴花的混合策略有多种

  - alpha加法混合
  - 覆盖混合

  
  
- 如何计算贴花的变换矩阵？

这一般通过射线检测来进行，针对射线集中点相关信息，来进行变换矩阵的计算。射线检测一般通过引擎的回调函数来触发。

# heightMapTracing 函数逐行解析

## 函数签名和目标
```glsl
float heightMapTracing(vec3 ori, vec3 dir, out vec3 p)
```

**输入参数**：
- `ori`: 光线起点（相机位置）
- `dir`: 光线方向（已归一化）
- `p`: 输出参数，交点位置

**返回值**：光线参数t，使得交点 = ori + dir * t

**核心任务**：寻找光线与动态海面高度场的精确交点

---

## 第1-2行：初始化搜索边界
```glsl
float tm = 0.0;        // 近端边界参数
float tx = 1000.0;     // 远端边界参数
```

**算法思想**：
- 建立搜索区间 [tm, tx]
- tm=0：从相机位置开始
- tx=1000：设定一个足够远的边界
- 这构成了二分搜索的初始区间

**几何意义**：
- 光线方程：P(t) = ori + dir * t
- 搜索范围：t ∈ [0, 1000]

---

## 第3行：远端边界检测
```glsl
float hx = map(ori + dir * tx);
```

**功能**：计算远端点的高度场值
- `ori + dir * tx`：计算t=1000处的3D坐标
- `map()`：返回该点相对于海面的高度差
  - 正值：点在海面上方（空气中）
  - 负值：点在海面下方（水中）

**数学含义**：
- map函数实际计算：f(x,y,z) = y - sea_height(x,z)
- 这是一个**隐式曲面方程**，零等值面就是海面

---

## 第4-7行：早期退出优化
```glsl
if(hx > 0.0) {
    p = ori + dir * tx;
    return tx;   
}
```

**逻辑判断**：如果远端点仍在海面上方
- **意味着**：整条光线都不会击中海面
- **优化策略**：直接返回远端点，避免无意义的搜索
- **性能收益**：对于天空像素，立即跳过复杂计算

**边界情况处理**：
- 适用于观察者高度很高的情况
- 或者光线方向朝向天空的像素

---

## 第8行：近端边界检测
```glsl
float hm = map(ori);
```

**功能**：计算起点（相机）的高度场值
- 通常相机在海面上方，所以hm > 0
- 这为后续的插值计算提供基础数据

**算法假设**：
- 起点在海面上方：hm > 0
- 远端在海面下方：hx < 0
- 保证了交点的存在性

---

## 第9行：主循环开始
```glsl
for(int i = 0; i < NUM_STEPS; i++) {
```

**循环控制**：
- NUM_STEPS = 32：最大迭代次数
- 平衡精度和性能
- 防止无限循环

---

## 第10行：智能中点估计（核心创新）
```glsl
float tmid = mix(tm, tx, hm / (hm - hx));
```

**这是最关键的一行！**

### 传统二分搜索 vs 改进算法

**传统二分搜索**：
```glsl
float tmid = (tm + tx) * 0.5;  // 简单中点
```

**本算法（割线法/Regula Falsi）**：
```glsl
float tmid = mix(tm, tx, hm / (hm - hx));
```

### 数学推导

**目标**：寻找更好的中点估计

假设高度函数在区间内线性变化：
```
h(t) ≈ hm + (hx - hm) * (t - tm) / (tx - tm)
```

求解 h(t) = 0（海面交点）：
```
0 = hm + (hx - hm) * (t - tm) / (tx - tm)
```

解得：
```
t = tm + (tx - tm) * (-hm) / (hx - hm)
t = tm + (tx - tm) * hm / (hm - hx)
```

这正是mix函数的定义：
```glsl
mix(tm, tx, hm / (hm - hx)) = tm + (tx - tm) * hm / (hm - hx)
```

### 几何直觉
```
      海面上方 hm > 0
         |
    tm --|-- tmid --|-- tx
         |           |
                海面下方 hx < 0
```

割线法利用函数值信息，选择更接近真实交点的位置，而不是盲目地选择几何中点。

---

## 第11-12行：计算中点信息
```glsl
p = ori + dir * tmid;     // 计算中点的3D坐标
float hmid = map(p);      // 计算中点的高度场值
```

**功能**：
- 根据新的参数tmid计算对应的3D位置
- 评估该位置相对于海面的高度关系

---

## 第13-18行：区间更新逻辑
```glsl
if(hmid < 0.0) {
    tx = tmid;      // 中点在海面下方
    hx = hmid;      // 更新上边界
} else {
    tm = tmid;      // 中点在海面上方  
    hm = hmid;      // 更新下边界
}
```

**二分搜索的核心逻辑**：

### 情况1：hmid < 0（中点在海面下方）
```
空气 |  空气  | 水中
tm --|-- tmid | tx
     |        |
   缩小搜索范围到 [tm, tmid]
```

### 情况2：hmid ≥ 0（中点在海面上方）
```
空气 | 空气  |  水中  
tm --|-- tmid|-- tx
     |       |
   缩小搜索范围到 [tmid, tx]
```

**不变量维护**：
- 始终保持：hm ≥ 0, hx < 0
- 确保交点始终在当前区间内

---

## 第19行：收敛判断
```glsl
if(abs(hmid) < EPSILON) break;
```

**收敛条件**：
- EPSILON = 1e-3：精度阈值
- |hmid| < ε 意味着非常接近海面
- 提前终止，避免过度计算

**数值分析考虑**：
- 避免无限精度追求
- 在视觉质量和性能间找平衡

---

## 第21行：最终插值（精度保证）
```glsl
return mix(tm, tx, hm / (hm - hx));
```

**最后一次插值**：
- 即使循环结束，仍用割线法估计最佳交点
- 确保返回值的连续性和精度
- 避免突然的精度跳跃

## 与其他算法的比较

| 算法类型 | 收敛速度 | 实现复杂度 | 数值稳定性 |
|----------|----------|------------|------------|
| 传统二分搜索 | O(log n) | 简单 | 高 |
| 牛顿法 | O(n²) | 需要导数 | 可能发散 |
| 割线法（本算法） | O(n^1.618) | 中等 | 高 |
| 步进法 | O(n) | 简单 | 易产生伪影 |

这个算法选择了割线法的变种，在实时渲染的约束下达到了最佳的效果平衡。

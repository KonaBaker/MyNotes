# getNormal 函数逐行深度解析

## 函数签名和核心任务
```glsl
vec3 getNormal(vec3 p, float eps)
```

**输入参数**：
- `p`: 海面上的一个点（已通过光线追踪找到）
- `eps`: 微分步长，控制数值微分的精度

**返回值**：该点处海面的单位法向量

**核心任务**：计算复杂动态海面在任意点处的表面法向量

### 隐式曲面的法向量理论
对于隐式曲面 F(x,y,z) = 0，法向量就是梯度：
```
n⃗ = ∇F = (∂F/∂x, ∂F/∂y, ∂F/∂z)
```

在我们的案例中：
```
F(x,y,z) = y - sea_height(x,z,t) = map(x,y,z)
```

所以法向量应该是：
```
n⃗ = (∂map/∂x, ∂map/∂y, ∂map/∂z)
```

但是**解析求导是不可能的**！因为map函数包含分形噪声、三角函数、非线性变换...

---

## 第1-2行：初始化和中心点采样
```glsl
vec3 n;
n.y = map_detailed(p);
```

### 为什么用map_detailed而不是map？
- `map()`: 用于光线追踪，迭代次数少（ITER_GEOMETRY = 3）
- `map_detailed()`: 用于精细计算，迭代次数多（ITER_FRAGMENT = 5）

**设计哲学**：
- 光线追踪阶段：速度优先，粗糙计算
- 着色阶段：质量优先，精细计算

### n.y的含义
这里`n.y`**暂时**存储中心点的高度场值，**不是最终法向量的y分量**！

这是一个聪明的变量复用技巧。

---

## 第3行：X方向的数值微分
```glsl
n.x = map_detailed(vec3(p.x+eps,p.y,p.z)) - n.y;
```

### 数学原理：前向差分
标准的偏导数定义：
```
∂f/∂x = lim(h→0) [f(x+h,y,z) - f(x,y,z)] / h
```

数值近似：
```
∂f/∂x ≈ [f(x+eps,y,z) - f(x,y,z)] / eps
```

但这里有个**关键简化**：分母eps被省略了！

**为什么可以省略eps？**
1. 最后会进行`normalize()`，比例因子会被消除
2. 只关心方向，不关心magnitude
3. 避免除法运算，提升GPU性能

### 几何意义
```
         高度
          ↑
          |
      p+h |  * (x+eps, y, z)
          | /
      p   |*  (x, y, z)
          |
          └──────────────→ x方向
```

`n.x`存储的是**X方向的高度变化率**（未归一化）

---

## 第4行：Z方向的数值微分
```glsl
n.z = map_detailed(vec3(p.x,p.y,p.z+eps)) - n.y;
```

**完全相同的逻辑**：
- 计算Z方向的高度变化率
- 同样省略了eps分母
- 利用之前存储的n.y（中心点值）避免重复计算

### 为什么不计算Y方向？
对于高度场 F(x,y,z) = y - h(x,z)：
```
∂F/∂x = -∂h/∂x
∂F/∂y = 1        ← 这是常数！
∂F/∂z = -∂h/∂z
```

**Y分量总是常数1**（或者考虑eps后是eps），所以不需要复杂计算。

---

## 第5行：设置Y分量
```glsl
n.y = eps;
```

### 为什么是eps而不是1？

**数学推导**：
如果我们完整地写出数值微分：
```
n.x = [map(p+Δx) - map(p)] / eps
n.y = 1 (理论值)
n.z = [map(p+Δz) - map(p)] / eps
```

为了保持一致性，n.y也应该具有相同的"量纲"：
```
n.y = eps / eps = 1 (在量纲上) ≈ eps (在数值上)
```

**实际意义**：
- 当eps很小时，n.y ≈ 0，法向量主要由水平分量决定（陡峭的波浪）
- 当eps较大时，n.y较大，法向量更接近垂直（平缓的海面）

这是一个**自适应法向量计算**的巧妙设计！

---

## 第6行：归一化
```glsl
return normalize(n);
```

**必要性**：
1. 确保法向量长度为1
2. 消除eps缩放因子的影响
3. 满足光照计算的数学要求

**数学效果**：
```
原始向量：n = (∂h/∂x, eps, ∂h/∂z)
归一化后：n̂ = n / |n|
```

## eps参数的深层含义

### eps的动态计算
在调用时：
```glsl
vec3 n = getNormal(p, dot(dist,dist) * EPSILON_NRM);
```

其中：
```glsl
#define EPSILON_NRM (0.1 / iResolution.x)
```

### 自适应精度系统

#### 1. **距离自适应**
- `dot(dist,dist)`: 距离平方，远处用大步长
- 近处：eps小，高精度，看得清细节
- 远处：eps大，低精度，避免噪声

#### 2. **分辨率自适应**  
- `iResolution.x`: 屏幕宽度
- 高分辨率：需要更高精度的法向量
- 低分辨率：粗糙法向量就足够了

#### 3. **数值稳定性**
```
eps太小 → 数值误差放大，产生噪声
eps太大 → 丢失细节，表面平滑化
```

**黄金平衡点**：eps = distance² × (0.1 / screen_width)

## 与其他方法的比较

### 1. **中心差分法**（更精确但更昂贵）
```glsl
vec3 getNormalCentral(vec3 p, float eps) {
    float dx = map(p + vec3(eps,0,0)) - map(p - vec3(eps,0,0));
    float dz = map(p + vec3(0,0,eps)) - map(p - vec3(0,0,eps));
    return normalize(vec3(-dx, 2.0*eps, -dz));
}
// 需要4次map调用，成本翻倍
```

### 2. **解析法向量**（理想但不可行）
```glsl
// 如果海面有解析表达式 h(x,z) = sin(x) + cos(z)
vec3 getAnalyticalNormal(vec3 p) {
    float dx = cos(p.x);    // ∂h/∂x
    float dz = -sin(p.z);   // ∂h/∂z
    return normalize(vec3(-dx, 1.0, -dz));
}
// 但我们的海面是分形的，没有解析表达式！
```

### 3. **本算法**（最佳权衡）
- **3次函数调用**：比中心差分省一半
- **前向差分**：精度略低但够用
- **自适应步长**：距离和分辨率感知
- **GPU友好**：简单计算，高并行性

## 算法的视觉影响

### 法向量质量对渲染效果的影响

#### 1. **高质量法向量**
```
光滑的波光粼粼效果
准确的反射
自然的高光分布
```

#### 2. **低质量法向量**  
```
锯齿状光照
不自然的反射
块状化的高光
```

### 为什么这个算法效果好？

#### 1. **多尺度细节保持**
- `map_detailed`使用5次分形迭代
- 保留了从大波浪到小波纹的所有细节
- 法向量包含丰富的频率信息

#### 2. **自适应精度**
- 近景：高精度，展现细腻的水面纹理
- 远景：适度简化，避免闪烁噪声
- 不同分辨率下都有良好表现

#### 3. **数值稳定性**
- eps的动态调整避免了常见的数值问题
- 归一化确保了法向量的一致性

## 性能分析

### 计算成本分解
```glsl
vec3 getNormal(vec3 p, float eps) {
    vec3 n;
    n.y = map_detailed(p);                           // 成本：100%
    n.x = map_detailed(vec3(p.x+eps,p.y,p.z)) - n.y; // 成本：100%  
    n.z = map_detailed(vec3(p.x,p.y,p.z+eps)) - n.y; // 成本：100%
    n.y = eps;                                       // 成本：~0%
    return normalize(n);                             // 成本：~1%
}
// 总成本：≈ 3 × map_detailed()调用
```

### 与其他shader模块的成本比较
- `heightMapTracing`: 32 × map() ≈ 32 × 60% = 1920%
- `getNormal`: 3 × map_detailed() ≈ 3 × 100% = 300%
- `光照计算`: ≈ 10%

**结论**：法向量计算虽然昂贵，但相对于光线追踪还是可接受的。

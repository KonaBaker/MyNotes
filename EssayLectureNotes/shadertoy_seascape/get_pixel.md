# getPixel 函数完整渲染管道解析

## 函数概览与渲染管道

glsl

```glsl
vec3 getPixel(in vec2 coord, float time)
```

**核心职责**：将屏幕像素坐标转换为RGB颜色值

**渲染管道流程**：

```
屏幕坐标 → 射线生成 → 光线追踪 → 表面着色 → 颜色混合 → 最终像素
    ↓         ↓         ↓         ↓         ↓         ↓
  coord   →   dir   →    p    →    n    →   色彩   →  RGB
```

这是一个**完整的光线追踪渲染器**在单个函数中的实现！

------

## 第一阶段：坐标标准化 (Coordinate Normalization)

glsl

```glsl
vec2 uv = coord / iResolution.xy;
uv = uv * 2.0 - 1.0;
uv.x *= iResolution.x / iResolution.y;
```

### 第1行：屏幕空间标准化

glsl

```glsl
vec2 uv = coord / iResolution.xy;
```

**数学变换**：

```
输入: coord ∈ [0, width] × [0, height]
输出: uv ∈ [0, 1] × [0, 1]
变换: (x, y) → (x/width, y/height)
```

**几何意义**：将像素坐标标准化到单位正方形。

### 第2行：NDC坐标转换

glsl

```glsl
uv = uv * 2.0 - 1.0;
```

**数学变换**：

```
输入: uv ∈ [0, 1] × [0, 1]
输出: uv ∈ [-1, 1] × [-1, 1]
变换: (u, v) → (2u-1, 2v-1)
```

**几何意义**：移动坐标原点到屏幕中心。

### 第3行：纵横比校正

glsl

```glsl
uv.x *= iResolution.x / iResolution.y;
```

**目的**：确保正方形在屏幕上显示为正方形，圆形显示为圆形。

**效果分析**：

```
16:9 屏幕: uv.x ∈ [-1.778, 1.778], uv.y ∈ [-1, 1]
4:3 屏幕:  uv.x ∈ [-1.333, 1.333], uv.y ∈ [-1, 1]
1:1 屏幕:  uv.x ∈ [-1, 1], uv.y ∈ [-1, 1]
```

**最终结果**：uv坐标表示相机的**视角空间**，原点在屏幕中心。

## 第二阶段：相机系统设置 (Camera System Setup)

### 相机动画

glsl

```glsl
vec3 ang = vec3(sin(time*3.0)*0.1,sin(time)*0.2+0.3,time);
```

**三轴旋转参数**：

- Pitch (俯仰角)

  ：

  ```
  sin(time*3.0)*0.1
  ```

  - 频率：3Hz（快速振动）
  - 幅度：±0.1弧度 ≈ ±5.7°
  - 效果：轻微的上下点头运动

- Yaw (偏航角)

  ：

  ```
  sin(time)*0.2+0.3
  ```

  - 频率：1Hz（缓慢摇摆）
  - 幅度：±0.2弧度 ≈ ±11.5°
  - 偏移：+0.3弧度 ≈ +17.2°
  - 效果：左右摇摆 + 整体向右倾斜

- Roll (滚转角)

  ：

  ```
  time
  ```

  - 频率：持续旋转
  - 效果：相机绕视线轴匀速旋转

**设计意图**：模拟在海上的**自然摇摆感**，增强沉浸体验。

### 相机位置

glsl

```glsl
vec3 ori = vec3(0.0,3.5,time*5.0);
```

**运动轨迹**：

- **X坐标**：0.0（始终在中心线）
- **Y坐标**：3.5（固定高度3.5米）
- **Z坐标**：time*5.0（5米/秒匀速前进）

**物理意义**：模拟在海面上方**匀速航行**的视角。

## 第三阶段：射线生成 (Ray Generation)

### 基础射线方向

glsl

```glsl
vec3 dir = normalize(vec3(uv.xy,-2.0));
```

**射线构造**：

- **X, Y分量**：来自屏幕坐标`uv.xy`
- **Z分量**：固定为-2.0（负Z表示"向前"）

**几何意义**：

```
        屏幕平面 (z = -2)
           |
           |     ∕ 射线方向
    Y↑     |   ∕ dir
      ↑    | ∕
      |    |∕
    ——————○——————————→ X
           相机原点
```

**视场角计算**：

```
FOV = 2 × arctan(aspect_ratio / 2)
对于16:9屏幕: FOV ≈ 69.4°
```

### 桶形畸变

glsl

```glsl
dir.z += length(uv) * 0.14;
```

**数学分析**：

```
r = length(uv) = √(uv.x² + uv.y²)
dir.z_new = -2.0 + 0.14 × r
```

**径向效应**：

- **屏幕中心** (r=0)：dir.z = -2.0（无变化）
- **屏幕边缘** (r≈2.5)：dir.z ≈ -1.65（Z分量增大）

**视觉效果**：

```
原始投影：平直的透视效果
畸变后：广角镜头的桶形效果
边缘像素：视角向外扩展
```

**物理意义**：模拟**鱼眼镜头**或**人眼的周边视觉**效果。

### 旋转变换

glsl

```glsl
dir = normalize(dir) * fromEuler(ang);
```

**数学操作**：

1. **重新归一化**：确保射线方向为单位向量
2. **旋转变换**：应用相机的欧拉角旋转
3. **最终射线**：在世界坐标系中的实际方向

**矩阵形式**：

```
dir_world = R_z(roll) × R_y(yaw) × R_x(pitch) × dir_camera
```

## 第四阶段：光线追踪 (Ray Tracing)

glsl

```glsl
vec3 p;
heightMapTracing(ori,dir,p);
```

### 函数调用分析

**输入参数**：

- `ori`：射线起点（相机位置）
- `dir`：射线方向（已旋转的单位向量）
- `p`：输出参数，存储交点位置

**核心算法**：使用改进的二分搜索法寻找射线与动态海面的交点。

### 追踪过程

1. **射线方程**：P(t) = ori + t × dir
2. **海面方程**：map(P(t)) = 0
3. **求解目标**：找到使 map(P(t)) = 0 的参数t
4. **交点计算**：p = ori + t × dir

**数学复杂性**：这是一个**非线性隐式方程**的求根问题，没有解析解。

------

## 第五阶段：表面几何计算 (Surface Geometry)

### 距离向量

glsl

```glsl
vec3 dist = p - ori;
```

**几何意义**：从相机到海面交点的向量。

**用途**：

- 计算观察距离：`length(dist)`
- 深度相关的效果（雾效、细节层次）
- 自适应精度控制

### 法向量计算

glsl

```glsl
vec3 n = getNormal(p, dot(dist,dist) * EPSILON_NRM);
```

**自适应精度**：

- `dot(dist,dist)`：距离的平方
- `EPSILON_NRM`：基础精度因子
- **总精度**：距离越远，微分步长越大

**数学原理**：

```
eps = distance² × (0.1 / screen_width)
近距离：高精度法向量，展现细腻波纹
远距离：低精度法向量，避免数值噪声
```

### 光源设置

glsl

```glsl
vec3 light = normalize(vec3(0.0,1.0,0.8));
```

**光源方向分析**：

- **X分量**：0.0（正面照射）
- **Y分量**：1.0（从上方照射）
- **Z分量**：0.8（略微倾斜）

**归一化后**：

```
light ≈ (0, 0.781, 0.625)
```

**物理意义**：模拟**午后阳光**，不是正午的垂直照射，而是略有角度，创造更好的立体感。

## 第六阶段：颜色计算与混合 (Color Computation & Blending)

glsl

```glsl
return mix(
    getSkyColor(dir),
    getSeaColor(p,n,light,dir,dist),
    pow(smoothstep(0.0,-0.02,dir.y),0.2));
```

### 混合组件分析

#### 1. 天空颜色

glsl

```glsl
getSkyColor(dir)
```

**输入**：射线方向（在海面之上的部分） **输出**：程序化生成的天空颜色 **用途**：为反射和背景提供环境光照

#### 2. 海水颜色

glsl

```glsl
getSeaColor(p,n,light,dir,dist)
```

**输入参数**：

- `p`：海面交点位置
- `n`：表面法向量
- `light`：光源方向
- `dir`：观察方向
- `dist`：距离向量

**计算内容**：

- 菲涅尔反射
- 漫射和镜面光照
- 次表面散射
- 深度相关的颜色变化

#### 3. 混合权重函数

glsl

```glsl
pow(smoothstep(0.0,-0.02,dir.y),0.2)
```

这是最精妙的部分！让我们深入分析：

### 混合权重的数学构造

#### 第1步：视线角度检测

glsl

```glsl
dir.y  // 射线的Y分量（垂直角度）
```

**几何意义**：

- `dir.y > 0`：射线向上，看向天空
- `dir.y = 0`：射线水平，看向地平线
- `dir.y < 0`：射线向下，看向海面

#### 第2步：平滑过渡函数

glsl

```glsl
smoothstep(0.0,-0.02,dir.y)
```

**函数行为**：

```
当 dir.y ≥ 0 (向天空):     返回 0
当 dir.y ≤ -0.02 (向海面): 返回 1  
当 -0.02 < dir.y < 0:     平滑过渡 0→1
```

**过渡区域**：仅在地平线附近2.3°的角度范围内！

#### 第3步：非线性调节

glsl

```glsl
pow(..., 0.2)
```

**数学效果**：

```
pow(x, 0.2) = x^0.2 = ⁵√x
这是一个"向上弯曲"的曲线
```

**视觉效果**：

- 让海水颜色更快地占主导
- 在地平线附近创造更柔和的过渡
- 减少"突然切换"的视觉伪影

### 混合逻辑的物理意义

这个混合函数实现了**基于视角的智能颜色选择**：

```
向上看 (dir.y > 0):
    权重 ≈ 0 → 主要显示天空色
    物理意义：射线没有击中海面，应该看到天空

向下看 (dir.y < -0.02):  
    权重 ≈ 1 → 主要显示海水色
    物理意义：射线击中海面，应该看到水面

地平线附近 (-0.02 < dir.y < 0):
    权重 0→1 平滑过渡
    物理意义：海天交界的自然融合
```

## 完整渲染流程示例

### 场景参数

```
像素坐标：(1200, 600) - 屏幕偏右上方
屏幕分辨率：1920×1080
时间：t = 10秒
```

### 第1步：坐标标准化

```
uv = (1200/1920, 600/1080) = (0.625, 0.556)
uv = (0.625*2-1, 0.556*2-1) = (0.25, 0.112)
uv.x *= 1920/1080 = 1.778
uv = (0.445, 0.112)
```

### 第2步：相机设置

```
ang = (sin(30)*0.1, sin(10)*0.2+0.3, 10)
    = (-0.099, 0.156, 10) 弧度
ori = (0, 3.5, 50)
```

### 第3步：射线生成

```
dir_camera = normalize(0.445, 0.112, -2.0) = (0.218, 0.055, -0.978)
r = length(0.445, 0.112) = 0.459
dir_camera.z += 0.459 * 0.14 = -0.978 + 0.064 = -0.914
dir_camera = normalize(0.445, 0.112, -0.914) = (0.437, 0.110, -0.893)
dir_world = fromEuler(ang) * dir_camera  // 复杂的旋转变换
```

### 第4步：光线追踪

```
通过heightMapTracing找到交点 p
假设结果：p = (某个x, 海面高度, 某个z)
```

### 第5步：表面计算

```
dist = p - ori
n = getNormal(p, ||dist||² * EPSILON_NRM)
light = (0, 0.781, 0.625)
```

### 第6步：颜色混合

```
由于dir.y < 0（向下看海面）：
mix_weight = pow(smoothstep(0.0, -0.02, dir.y), 0.2) ≈ 1.0

最终颜色 ≈ getSkyColor(dir) * 0.0 + getSeaColor(...) * 1.0
          = getSeaColor(...)
```

**结果**：这个像素显示海水颜色。
